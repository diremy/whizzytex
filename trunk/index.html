WhizzyTeX

What i zee iz what you TeX

This achive whizzytex.tgz provides an emacs minor mode and a shell-script deamon for incrementally previewing TeX files (configured to work either with ghostscript-based or dvi-based previewers).

The archive contains the following files: 

        whizzytex
        whizzytex.el
        whizzytex.sty
        Makefile.tpl
        manual.tex
        slides.tex
        CHANGES
The three first files are enough to run WhizzyTeX. They can all be in the current working directory (but can be installed in more appropriate places). The file whizzytex is a shell-script and should be executable.

The files manual.tex and slides.tex are two examples that you can explore with WhizzyTeX. The latex source file manual.tex is a self-demonstrating documentation. The file Makefile.tpl is a template to illustrate the use of the command make as a preprocessor to LaTeX-ing and is used in the example manual.tex (it is definitely not a makefile for installing WhizzyTeX). The file CHANGES list the most recent changes to the WhizzyTeX.

Documentation

The file artcile.tex is actually a short documentation that you can explore interactively with WhizzyTeX or read in DVI or HTML format.

Disclaimer

WhizzyTeX is still under development and may contain dangerous bugs. We recommend you to save copies of the file (and of the whole directory you are working in) before WhizzyTeX-ing.

Previewers

One of the problem that you will encounter with WhizzyTeX, especially in if you have a relatively old slow machine is that the previewer flashes whenever refreshing the page. xdvi is much faster but does not render the color while gv is much slower. Both flashes at each refreshing of the page. We suggest two alternatives:
  1. gv

    Fabrice Le Fessant has kindly patched the version 3.5.8 of GV so as to avoid this problem, drawing in a pixmap and flushing the pixmap all at once. Here are the patched version (and its original)

    To remove flashing, you must then call gv with the option -nopixmap.

  2. advi (Active Dvi)

    This is a DVI-previewer, written by Alexandre Miquel and extended by Jun Furuse and myself Furuse.

    It is alike xdvi, but written in Ocaml. However, it handles colors and Postscript specials better than xdvi. Moreover, it recognizes a new kind of specials specials (mldvi) that kind be used to make pauses, record some and replay a sequence of xdvi.

    More importantly, by default, advi draw the page in the background and flushes the page altogether, which avoids flashing effect when redrawing a page that is almost unchanged. This a requirement for WhizzyTeX, but also quite useful for display slides with overlays.

    See the Active Dvi distribution (or pick the distribution under CVS untill version 0.4 or higher are made available) Good options for "advi" are 

    advi -nowatch -html Start-Document OTHER-OPTIONS FILENAME
